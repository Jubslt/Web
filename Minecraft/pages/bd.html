Banco de Dados

Banco de Dados

Bancos de dados são essenciais em sistemas de software modernos, permitindo o armazenamento, organização e recuperação eficiente de grandes volumes de dados. Esta introdução explora os conceitos básicos, vantagens e tipos comuns de bancos de dados.

Banco de dados relacionais

Bancos de dados relacionais armazenam dados em tabelas organizadas, usando SQL para gerenciar e manipular dados. Sendo assim, cada tabela criada tem uma chave primária única e pode se relacionar com outras tabelas por meio de chaves estrangeiras.

Diagrma de ER

O Diagrma de ER é uma representação gráfica que mostra a estrutura lógica de um banco de dados. Desta forma, ilustrando as entidades relevantes, seus atributos e os relacionamentos entre elas.

°PRIMARY KEY:valor único e não ausente que representa cada item salvo no Banco de Dados.

°FOREIGN KEY: é a primary key importada de outra entidade que estabelece relacionamento entre as entidades.

°REGRA: Em um relacionamento 1 para N, a foreign key fica sempre do lado N.

O que é SQL e quais suas principais funções e componentes

SQL é uma linguagem padrão para gerenciar e manipular bancos de dados relacionais, permitindo operações como inserção, atualização, exclusão e consulta de dados.

Principais funções:

Consulta de Dados;

Inserção de Dados;

Atualização de Dados;

Exclusão de Dados;

Criação e Modificação de Estrutura de Banco de Dados.

Componentes:

DDL (Data Definition Language):

Define a estrutura do banco de dados, como tabelas e índices.

Comandos: CREATE, ALTER, DROP

DML (Data Manipulation Language):

Manipula os dados dentro das tabelas.

Comandos: SELECT, INSERT, UPDATE, DELETE.

DQL (Data Query Language):

Envolve comandos para consultar e recuperar dados de um banco de dados.

O principal comando utilizando é o SELECT.

DTL (Data Transaction Language):

Refere-se ao controle de transações, garantindo a integridade dos dados durante operações complexas.

Comando: COMMIT, ROLLBACK, e SAVEPOINT.
